已知模型结构的 ONNX 模型转 PyTorch 的新思路

缘由

现在网络上有些"开源"模型, 模型架构就是直接 import xxx from transformers, 训练过程呢, 暂不开源, 就在 Hugging Face 上面丢个反向优化过的 model.onnx, 然后再开个知识星球或者 B 站工坊, 卖他的 Pro 版.

甚至 Pro 版也只卖给你 ONNX, 想要 PyTorch? 想微调? 可以, 来买我一千五百刀的企业 Pro Plus 版.

我觉得这种"开源"已经和开源精神没什么关系了, 这完全就是为了吃开源的流量红利然后卖没反向优化过的版本而已. 哦, 顺便还有挂网上个前端拿你的数据做训练.

那我们肯定不能便宜了他, 正好你的模型架构不是直接调包吗, 那我有的是办法把你逆出来.

思考

现在网络上的绝大多数 ONNX 转 PyTorch 模型的工具都是针对通用的模型的, 但是由于我们的目的是逆向一个架构已知, 而且是直接调包的模型, 那我们自然可以用一些通用工具用不了的思路.

观察

经过对其发布的 ONNX 模型和 Torch 模型(随机初始化出来的) 的观察, 我发现:

  • 两个模型的权重数量相近, 具体来说, 是 Encoder 部分差2, 而 Decoder 部分相等.

  • ONNX 中的权重大多数都有可读的名字, 但是少数的名字变成了如 onnx::MatMul_1234 这种不可读的形式.

这里比较麻烦的就是这些名字丢失的权重, 不过因为权重的数量是相近的, 我还是猜测这些权重之间是一一对应的.

另外由于部分权重的名字仍然可见, 我猜测该作者是用一些比较官方的手段把 PyTorch 模型导出为 ONNX 模型的, 在我的例子中, 具体来说, 就是

这个工具还是比较友好的, 他转出来的 ONNX 都是 consistent 的, 这就方便了我们更直接地得知导出过程中到底发生了什么.

实验

我首先设法构造了这样的一个模型, 它的权重是以 0.1 递增的数字, 然后将其导出为 ONNX.

这时我发现 ONNX 模型中名称不可读的权重果然都与 PyTorch 中的某些权重对应, 而前面提到的差 2 的权重则是与池化相关的, 想必是没有开启, 因此无关紧要, 而且更加印证了我的想法.

但是有一个问题是, 这些名字丢失的权重, 虽然内部的值(全都是0.x)对得上 PyTorch 模型中的某个权重, 但是有一部分的 shape 却对不上, 具体来说, 是一些矩阵的大小转置了.

为了确认大小转置了是不是里面的值也确实转置了, 我又构造了另一个模型, 这个模型中的权重都是不对称的, 第一行全都是某个值, 其余行全都是另一个值, 这两个值递增且不想等.

这次的结果确认了我的想法, 那些名字丢失的矩阵就是被转置了. 关于为什么要转置, 我想可能是因为某种优化:

AB=(BTAT)TAB = \left(B^T A^T\right)^T AB=(BTAT)T

总之我确实找到了 ONNX 和 PyTorch 模型的权重之间的对应关系.

结果

结果证明了我的想法是对的. 根据这个原理我编写了一个转换程序, 转换得到的模型与原模型的行为完全一致.

转换程序已开源, 我并没有把这个写成一个非常通用的工具, 只是以本文的这个例子走一遍流程, 如果真的有同样的需求应该也能基本复制.

那么我为什么要逆向这个模型呢? 肯定不是纯粹为了给作者找不痛快, 其实是因为我在做一个 OCR 模型, 需要用他的底模微调.

总之我还是希望大家能多搞点真正的开源. 把数据集, 训练代码, 用的超参数, 都公开出来让别人看看, 如果这样就赚不到钱了, 那我建议你完全闭源, 也别天天发视频都带个"[开源]", 宣传的时候都用"开源模型最强", “比肩商业模型”, 反正你那开源约等于什么都没开.
